快速搭建自己的静态博客 - hexo

hexo 基于nodejs快速搭建自己的博客.
参考了官方文档和其他人的blog,很快把自己的博客搭好了. 而且不用申请服务器,直接静态页面托管在github上,简直不要太爽^_~!

准备条件

1. 安装nodejs , git
2. 安装hexo-cli 
       npm install -g hexo-cli

初始化项目(建站)

$ hexo init myBlog      
$ cd myBlog      
$ npm install

目录结构

├── _config.yml    //网站的 配置 信息，您可以在此配置大部分的参数。
├── package.json
├── scaffolds       //模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。
├── source          //资源文件夹是存放用户资源的地方
 |   ├── _drafts
 |   └── _posts
└── themes       //主题 文件夹。Hexo 会根据主题来生成静态页面。

配置文件中的各项配置不再赘述,官方文档写的很详细.

开始写作

新建一篇文章 

$ hexo new [layout] welcome

[layout]是布局参数,默认为post可以不加此参数,修改在_config.yml”的default_layout中.

档是markdown格式的, 需要按照markdown语法写.如下:

生成静态文件

$ # 可以使用简写方式：hexo g , 最终文件会生成到public文件夹下(没有会自动创建)
$ hexo generate

本地预览

执行一下命令，在浏览器中打开：http://localhost:4000/ 进行预览

$ # 简写：hexo s
$ hexo server
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

准备发布到github

在自己的github上建一个仓库, 名字格式 xxx.github.io ,” xxx” 是你的博客名字
在仓库下面新建一个index.html, 里面随便写点东西. 然后访问 https://xxx.github.io/ 看效果.
如果可以正常访问ok,接下准备部署自己的静态博客.

1.安装 hexo-deployer-git

$ npm install hexo-deployer-git

2.修改 _config.yml 配置

deploy:
  type: git
  repo: https://github.com/Sanonz/your-blog.github.io.git
  branch: master
  # message: [message]    #这是git默认提交的说明.

Hexo 一键部署到 GitHub

$ hexo deploy
部署成功后查看站点：https://your-blog.github.io

至此,一个简单漂亮的博客就完成了.

每次有新文章要发布,
就在本地项目里写,完成后发布到github上.

$ hexo new title.md
...写作
$ hexo g 
...生成静态页面
$ hexo s
...本地预览
$ hexo deploy
...部署,完成.

插曲: 在我写完这篇博客后,生成静态页面(hexo g) 时报错了.
"Error: Unable to call `now`, which is undefined or falsey"
原因:
这类异常一般是文章中使用了now( ), 小括号( )属于特殊字符,在编译文章时将now( )当函数处理了,结果找不到函数,就报错了。
解决:
可以参考Markdown语法第(0)章-特殊字符将now( )的小括号通过&#40; &#41; 进行转换
